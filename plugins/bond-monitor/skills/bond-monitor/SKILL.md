# 可转债申购监控技能

## 概述

此技能帮助用户自动监控可转债申购机会，通过 AkShare 获取数据，并通过 Server酱 推送到微信。

## 触发条件

当用户询问以下问题时自动触发：

- "今天有什么新债申购吗？"
- "查询可转债申购信息"
- "查看新债"
- "明日可转债"
- "债券申购提醒"
- "最近有什么可转债发行"

## 核心能力

### 1. 数据查询

- 使用 AkShare 库获取可转债数据
- 筛选今日和明日的可申购新债
- 提取关键信息：债券代码、名称、申购日期、转股价值、发行规模

### 2. 智能推送

- 通过 Server酱 推送到微信
- 支持自定义推送时间
- 支持每日状态通知（即使没有新债）
- 格式化推送内容，包含详细申购信息

### 3. 数据分析

- 计算转股溢价率
- 评估申购价值
- 提供投资建议

## 配置参数

### 必需参数

```yaml
serverchan_sendkey: SCT321629TV9CvNbMDefg
```

### 可选参数

```yaml
send_daily_status: false  # 是否发送每日状态通知
push_time: "09:00"        # 推送时间（24小时制）
```

## 使用示例

### 示例 1：基础查询

```
用户：今天有什么新债申购吗？

助手：让我为你查询今日的可转债申购信息...

[执行查询脚本]

📊 查询结果：
- 今日可申购：2 只
- 明日可申购：1 只

已推送到你的微信，请注意查收！
```

### 示例 2：配置推送时间

```
用户：帮我在每天早上 9 点推送新债信息

助手：好的，我来配置定时推送...

已配置：
- 推送时间：每天 09:00
- 推送渠道：微信（Server酱）
- 推送内容：当日和次日可申购的新债信息

配置生效后，你会在每个交易日早上收到推送。
```

### 示例 3：启用每日状态

```
用户：即使没有新债也每天提醒我

助手：好的，我来启用每日状态通知...

已配置：
- send_daily_status: true
- 即使没有新债也会发送"今日无新债申购"的提醒

这样可以确保你不会错过任何申购机会。
```

## 工作流程

### 查询流程

1. **获取数据**：调用 AkShare API 获取可转债数据
2. **筛选数据**：提取今日和明日的申购信息
3. **分析数据**：计算关键指标（转股价值、溢价率）
4. **推送通知**：通过 Server酱 发送到微信

### 定时任务流程

1. **定时触发**：在指定时间自动执行
2. **查询数据**：获取最新可转债信息
3. **判断推送**：
   - 有新债：推送详细信息
   - 无新债且启用每日状态：推送"无新债"提醒
   - 无新债且未启用每日状态：不推送

## 错误处理

### 常见错误

1. **网络错误**
   ```
   ❌ 获取数据失败：网络连接超时
   请检查网络连接后重试
   ```

2. **API 错误**
   ```
   ❌ AkShare API 调用失败：[错误信息]
   可能是服务器繁忙，请稍后重试
   ```

3. **推送错误**
   ```
   ❌ 推送失败：SendKey 无效
   请检查 Server酱 配置
   ```

### 错误恢复

- 自动重试机制（最多 3 次）
- 详细的错误日志记录
- 提供错误解决建议
